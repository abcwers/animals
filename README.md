<h2>Gyvūnėlių prieglaudos aplikacija</h2>
Versija 0.1
<h3>Apie aplikaciją</h3>
Gyvūnėlių prieglaudos aplikacija skirta efektyviai surasti ir įvesti globotinus gyvūnus. Kiekvienas aplikacijos vartotojas turi galimybę realiu laiku ieškoti esamų ir įvesti naują globotiną gyvūną.

<h2>Funkciniai reikalavimai</h2>
<p>Naudotojai ir rolės</p>

- [ ] **Administratorius** – atsakingas už aplikacijos nustatymus.
- [x] **Vartotojai** – turi teisę naudotis aplikacija. (SQL DB numatėme vartotojus, puslapiu gali naudotis visi, kaip SVEČIAS)

**FR0**. Administratorius gali valdyti vartotojus ir gyvūnų kategorijas*

Gyvūnų kategorijas galima:

- [ ] 1. Sukurti – nurodant pavadinimą (pvz. Roplys, Katė, Šuo ir t.t.)
- [ ] 2. Redaguoti
- [ ] 3. Šalinti.

Vartotojus galima:
- [ ] 1. Sukurti (vartotojo vardas, el. paštas, slaptažodis)
- [ ] 2. Redaguoti
- [ ] 3. Šalinti.

**FR1**. Vartotojai gali prisijungti prie aplikacijos*

Aplikacijoje yra galimybė:
- [ ] 1. Registruotis (vartotojo vardas, el. paštas, slaptažodis)
- [ ] 2. Prisijungti (el.paštas, slaptažodis)

**FR2**. Vartotojas gali suvesti gyvūnėlius
- [x] 1. Suvesti naują gyvūną - nurodant vardą, kategoriją, amžių dokumentacijos buvimą. (forma su javascript tikrinimo taisyklėmis sukūrė [@indrerz](https://github.com/indrerz), [@abcwers](https://github.com/abcwers) pritaikė CSS taisykles prie dizaino, parašė formos laukų klaidų tikrinimo taisykles naudojant PHP juos išryškinant, po klaidos gavimo išsaugojo formos reikšmes, pritaikė failo pridėjimo scriptą, integravo su SQL DB)
- [ ] 2. Redaguoti savo suvestus gyvūnus*

**FR3**. Vartotojas gali matyti ir filtruoti gyvūnų sąrašą
- [x] 1. Matyti visų gyvūnų sąrašą. ([@abcwers](https://github.com/abcwers))
- [x] 2. Ieškoti gyvūnų pagal kategoriją. ([@abcwers](https://github.com/abcwers))
- [x] 3. Ieškoti gyvūnų pagal vardą ir amžių*. ([@abcwers](https://github.com/abcwers))

<p>
  
**FR4**. Vartotojas gali eksportuoti visų gyvūnų sąrašą CSV formatu.
  - [x] atlikta. (PHP scriptą pritaikė [@VilteBort](https://github.com/VilteBort))

<h2>Nefunkciniai reikalavimai</h2>

NFR1. Sistemos duomenys saugomi reliacinėje duomenų bazėje
- [x] atlikta. (duomenų bazę suprojektavo [@edma11](https://github.com/edma11), duomenimis užpildė [@VilteBort](https://github.com/VilteBort), [@abcwers](https://github.com/abcwers) į .sql failą pridėjo vartotojo, ir DB sukūrimo SQL komandas)

NFR2. Visi veiksmai sistemoje kaupiami įvykių žurnale*
- [ ] Visi veiksmai sistemoje kaupiami įvykių žurnale. Kiekvienas įvykis turi registruoti įvykio laiką, įvykį ir reikalingą susijusią informaciją.

**NFR3**. Kartu su sistema pateikiama sistemos diegimo ir naudojimo dokumentacijos
- [x] 1. Naudojimo dokumentacija aprašo pagrindinių sistemos funkcijų panaudojimą. ([@VilteBort](https://github.com/VilteBort))
- [x] 2. Diegimo dokumentacija aprašo, kaip įdiegti ir paleisti sistemą. ([@VilteBort](https://github.com/VilteBort))

**NFR4**. Visi veiksmai sistemoje turi atsakyti greičiau nei per 2 sekundes
- [ ] Reikalavimas taikomas laikantis prielaidos, kad sistemoje yra iki 1000 gyvūnėlių įrašų. (netestuota su 1000 gyvūnėlių įrašų)
